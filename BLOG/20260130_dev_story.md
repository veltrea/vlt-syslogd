# Google Antigravity と Rust で Windows 用 Syslog サーバーを爆速で作った話

## はじめに：なぜ作ったのか
Windows 環境で手軽に使えて、かつ日本語（UTF-8）の表示にきちんと対応している Syslog サーバーが見当たらなかった、というのがすべての始まりです。
「無いなら作ればいい」の精神で、今回は **Google Antigravity** を使って生成してみました。

## Rust 言語の「生成適性」の高さ
今回、コアとなる実装はほぼ AI による「ワンショット生成」で動くものができました。
ここで **Rust 言語** を選択したのが勝因だったように思います。

なぜ Rust なのか？
1.  **GUI ライブラリの程よいシンプルさ**: Rust の GUI (eframe/egui) は機能が多すぎず、構造が明確です。Qt のような重厚長大なライブラリだと、ボイラープレートだけでも膨大になり、AI が一発で完動品を吐き出すのは難しかったでしょう。
2.  **歴史の浅さが逆に武器に**: Python など歴史が長すぎる言語は、古い書き方から最新の書き方まで「地層」のようにノウハウが混在しており、AI が混乱して古いコードを提案してくることがあります。Rust は比較的モダンな書き方に収束しており、AI が「素直で綺麗なコード」を書いてくれやすい傾向にあります。

ちょっとしたミニツールを作るとき、Rust は非常に強力な選択肢になると実感しました。

## 苦労した点：アイコン生成の泥沼
実装自体はスムーズでしたが、実は一番時間がかかった（10時間くらい！）のは **アプリケーションアイコン (.ico) の生成** でした。

最初は Google Antigravity に「高度な画像処理スクリプト」を書かせて Rust だけで完結させようとしたり、大きな画像をリサイズして...と頑張っていたのですが、全くうまくいかず大苦戦。
最終的に「Python スクリプトで変換すればいいのでは？」と切り替えて指示を出したところ、**ものの数分で解決** してしまいました。

## 教訓
AI の便利さは**「アプローチの仕方」と「指示の出し方」**で天と地ほどの差が出ます。
「AI に全部任せて複雑なことをやらせる」よりも、「AI が得意なシンプルな方法（今回は Python script）に誘導する」ことの重要さを痛感したプロジェクトでした。
